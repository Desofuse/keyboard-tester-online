How to Test a Keyboard
Fast checks for every key and common problems
Open tester All guides

If your keyboard feels “off” (missing letters, double input, weird shortcuts), do a quick full test. The online tester highlights each pressed key and shows the technical codes.

Start keyboard test If a key fails

1) Test letters and numbers

  1. Open the tester.
  2. Press A–Z slowly, then 0–9.
  3. Look for keys that don’t light up or light up late.

Tip: A key that triggers twice can be a switch issue or debris under the keycap.

2) Test modifiers (Shift, Ctrl, Alt, Win/Command)

Hold each modifier for a second to confirm it registers reliably. Then test combinations like Ctrl+C, Ctrl+V, and Alt+Tab.

  • Windows: Ctrl, Alt, Win
  • Mac: Control, Option, Command

3) Test special keys

  • Backspace, Enter, Space
  • Arrow keys + Home/End/PageUp/PageDown
  • F-keys (F1–F12) and Fn combinations (if your keyboard has Fn)
  • Media keys (volume, play/pause) if available

4) Check for ghosting / rollover (optional)

Ghosting happens when certain key combinations cannot be detected at the same time. If you play games or type fast, test common combos you use.

FAQ

Does this tester record my typing?

No. The tool reads keyboard events in your browser to highlight keys. It does not send your keystrokes anywhere.

Why do I see “event.code” and “keyCode”?

They help debugging. “event.code” describes the physical key position; “keyCode” is a legacy number used by older scripts.

One key works in the tester but not in my app — why?

It can be the app shortcut settings, language layout, accessibility tools, or a stuck modifier key.
